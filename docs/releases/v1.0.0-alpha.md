# v1.0.0-alpha

Release date: 2026-03-15

This alpha marks the point where the repository is no longer just a governance-document collection.

It is now a runnable AI coding runtime-governance framework prototype with:

- a real runtime lifecycle
- proposal-time guidance
- evidence-aware validation
- reviewable change-control artifacts
- an external domain contract seam already exercised across low-level domains

## Included In This Alpha

### Runtime Governance Spine

- `session_start`
- `pre_task_check`
- `post_task_check`
- `session_end`
- memory curation / promotion pipeline

## What This Alpha Is Good For

- teams that want AI coding governance with explicit plan/rule/risk context
- repos that need reviewable AI session artifacts instead of prompt-only interactions
- early adopters exploring domain-specific governance for firmware, drivers, or other high-context codebases

## Known Limits

This alpha is still intentionally bounded.

It does **not** yet provide:

- fully closed workflow interception across every IDE / edit / commit path
- hard-stop policy enforcement for most domain validators
- deep semantic reasoning comparable to a true policy engine
- a full plugin marketplace or versioned plugin API surface

## Recommended Evaluation Path

1. Install dependencies with `pip install -r requirements.txt`
2. Run `python governance_tools/quickstart_smoke.py --project-root . --plan PLAN.md --contract examples/usb-hub-contract/contract.yaml --format human`
3. Run `python governance_tools/example_readiness.py --format human`
4. Run `bash scripts/verify_phase_gates.sh`
