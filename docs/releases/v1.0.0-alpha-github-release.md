# v1.0.0-alpha GitHub Release Draft

`ai-governance-framework` has reached its first alpha as a runnable AI coding runtime-governance framework prototype.

## Highlights

- Runtime governance spine is operational across `session_start`, `pre_task_check`, `post_task_check`, and `session_end`.
- Proposal-time guidance, evidence-aware validation, and review-facing change-control artifacts are now part of the normal framework story.
- External domain contract seams are exercised across three low-level domains:
  - `USB-Hub-Firmware-Architecture-Contract`
  - `Kernel-Driver-Contract`
  - `IC-Verification-Contract`
- Trust-signal publishing now supports both:
  - CI/artifact-style bundle output
  - stable repo-local generated status under `docs/status/generated/`
- Release-package publishing now also supports a stable repo-local generated path under `docs/releases/generated/`

## Known Limits

This release is still intentionally bounded.

- workflow interception coverage is partial, not fully closed
- most domain enforcement remains mixed or advisory-first rather than universally hard-stop
- semantic verification is improving but still shallower than a full policy engine
- the external contract seam is real, but not yet a versioned plugin marketplace
